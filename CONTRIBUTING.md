# Contributing to remote-workstreams

The contributor workflow, testing gates, local voice guidance, and pull-request checklist live in [docs/contributing.md](docs/contributing.md). Start there, then read [docs/development.md](docs/development.md) and [docs/architecture.md](docs/architecture.md).

Please do not commit secrets, Keychain exports, agent transcripts, SQLite state, `.venv`, or downloaded model files.
